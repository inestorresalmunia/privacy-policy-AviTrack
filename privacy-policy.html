Política de Privacidad - AppPondex

Última actualización: Septiembre 2025

1. Información General

AviTrack es una aplicación diseñada para el escaneo de códigos QR y códigos de barras con validación de ubicación GPS. Esta política explica cómo utilizamos los permisos solicitados.

2. Permisos Solicitados y su Uso

📷 Permiso de Cámara

¿Por qué lo necesitamos? Para capturar imágenes de códigos QR y códigos de barras.

¿Cómo lo usamos? Solo se activa cuando el usuario presiona el botón de escanear. Las fotos se procesan localmente y se guardan temporalmente en el dispositivo.

Datos recopilados: Imágenes de códigos escaneados con timestamp de captura.

📍 Permisos de Ubicación (GPS)

¿Por qué lo necesitamos? Para validar que el usuario esté en la ubicación correcta según las coordenadas del código QR escaneado.

¿Cómo lo usamos? Se obtiene la ubicación actual solo cuando se escanea un código, comparándola con las coordenadas embebidas en el código QR.

Datos recopilados: Coordenadas GPS (latitud, longitud), precisión y timestamp de ubicación.

💾 Almacenamiento

¿Por qué lo necesitamos? Para guardar temporalmente las imágenes escaneadas y los datos de sesión.

¿Cómo lo usamos? Los datos se almacenan localmente en formato JSON hasta que se envían al servidor.

Datos almacenados: Imágenes procesadas, identificaciones de códigos, coordenadas y validaciones.

🌐 Internet

¿Por qué lo necesitamos? Para enviar los datos recopilados al servidor central y obtener direcciones a partir de coordenadas.

¿Cómo lo usamos? Se envían datos en lotes al servidor configurado. También se utiliza para convertir coordenadas GPS en direcciones legibles.

3. Tratamiento de Datos

  • Recopilación: Solo se recopilan datos necesarios para la funcionalidad de la aplicación.
  • Almacenamiento: Los datos se guardan localmente hasta su envío al servidor.
  • Transmisión: Los datos se envían de forma segura al servidor configurado.
  • Retención: Los datos locales se eliminan después del envío exitoso.

4. Seguridad

Implementamos medidas de seguridad para proteger sus datos:

  • Identificación única del dispositivo para evitar duplicados
  • Validación de datos antes del envío
  • Eliminación automática de datos locales tras el envío
  • Procesamiento local de imágenes

5. Datos que NO Recopilamos

  • Contactos o información personal del dispositivo
  • Historial de navegación
  • Otras aplicaciones instaladas
  • Comunicaciones privadas
  • Datos biométricos

6. Derechos del Usuario

El usuario puede:

  • Denegar permisos (la aplicación informará sobre funcionalidades limitadas)
  • Revisar los datos guardados localmente
  • Limpiar todos los datos de la aplicación desde la configuración del dispositivo
  • Desinstalar la aplicación en cualquier momento
